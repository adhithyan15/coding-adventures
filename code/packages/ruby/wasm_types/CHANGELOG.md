# Changelog

All notable changes to this package will be documented in this file.

## [0.1.0] - 2026-03-23

### Added

- Initial package scaffolding generated by scaffold-generator
- `VALUE_TYPE` frozen hash with all four WASM 1.0 numeric type codes:
  `i32` (0x7F), `i64` (0x7E), `f32` (0x7D), `f64` (0x7C)
- `BLOCK_TYPE_EMPTY` constant (0x40) for empty-result blocks
- `EXTERNAL_KIND` frozen hash with `function` (0x00), `table` (0x01),
  `memory` (0x02), `global` (0x03)
- `FUNCREF` constant (0x70) for the table element type
- `FuncType` Struct — function type signatures with params and results arrays
- `Limits` Struct — min/max page/element bounds for memories and tables
- `MemoryType` Struct — linear memory type wrapping Limits
- `TableType` Struct — funcref table type with element_type and limits
- `GlobalType` Struct — value_type plus mutable flag
- `Import` Struct — module_name, name, kind, and type_info
- `Export` Struct — name, kind, and index into index space
- `Global` Struct — global_type plus raw init_expr bytes
- `Element` Struct — table initializer: table_index, offset_expr,
  function_indices
- `DataSegment` Struct — memory initializer: memory_index, offset_expr, data
- `FunctionBody` Struct — expanded locals list plus raw code bytes
- `CustomSection` Struct — named arbitrary byte payload
- `WasmModule` class — mutable container for all twelve section types,
  constructor initializes all fields to empty arrays and `start` to nil
- 61 unit tests, all passing with 0 failures
- Literate programming style: ASCII diagrams, binary encoding tables,
  worked examples, and truth tables inline with the code
