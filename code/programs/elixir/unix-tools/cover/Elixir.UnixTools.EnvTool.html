5 ## What This Program Does
6
7 This is a reimplementation of the GNU `env` utility in Elixir. It prints
8 the current environment or runs a command with a modified environment.
9
10 ## How env Works
11
12 With no arguments, env prints all environment variables:
13
14 env => HOME=/Users/alice
15 PATH=/usr/bin:...
16 SHELL=/bin/zsh
17
18 With arguments, env can modify the environment before running a command:
19
20 env FOO=bar bash => runs bash with FOO=bar added
21 env -i bash => runs bash with an empty environment
22 env -u HOME bash => runs bash without the HOME variable
23
24 ## Modifier Flags
25
26 | Flag | Effect |
27 |------|-------------------------------------------------|
28 | -i | Start with a completely empty environment |
29 | -u | Remove (unset) a specific variable |
30 | -0 | End output lines with NUL instead of newline |
31 | -C | Change working directory before running command |
32
33 ## Argument Parsing
34
35 env uses a unique argument style. Arguments that contain `=` are treated
36 as environment variable assignments. The first argument without `=` starts
37 the command:
38
39 env A=1 B=2 my_command arg1 arg2
40 ^^^^^^^^^^^ ^^^^^^^^^^^^^^^^^^
41 assignments command + its args
42
43 ## Implementation Approach
44
45 1. `parse_assignments/1` separates NAME=VALUE pairs from the command.
46 2. `build_environment/3` constructs the final environment map.
47 3. `format_environment/2` formats env vars for display.
