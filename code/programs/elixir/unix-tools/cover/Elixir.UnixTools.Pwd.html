10 ## How CLI Builder Powers This

15
16 1. We hand `pwd.json` and `argv` to CLI Builder's `Parser.parse/2`.
17 2. The parser validates the input, enforces mutual exclusivity of
18 `-L` and `-P`, generates help text, and returns a tagged result.
19 3. We pattern-match on the result and run the business logic.
20
21 The result is that *this file contains only business logic*. All parsing,
22 validation, and help generation happen inside CLI Builder, driven by the
23 JSON spec.
24
25 ## Logical vs Physical Paths
26
27 When you `cd` through a symbolic link, the shell updates the `$PWD`
28 environment variable to reflect the path *as you typed it* — including
29 the symlink. This is the "logical" path.
30
31 The "physical" path resolves all symlinks. For example, if `/home` is
32 a symlink to `/usr/home`:
33
34 Logical: /home/user (what $PWD says)
35 Physical: /usr/home/user (what the filesystem says)
36
37 By default (`-L`), we print the logical path. With `-P`, we resolve
38 symlinks and print the physical path.
39
40 ## POSIX Compliance Note
41
42 If `$PWD` is not set, or if it doesn't match the actual current
43 directory, even `-L` mode falls back to the physical path. This
44 matches POSIX behavior.

140

147 The `$PWD` variable could be stale if:
