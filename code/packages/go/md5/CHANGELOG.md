# Changelog

All notable changes to this package will be documented in this file.

## [0.3.0] - 2026-03-31

### Changed

- **Operations system integration**: All public functions and methods (`SumMD5`,
  `HexString`, `New`, `Digest.Write`, `Digest.SumMD5`, `Digest.HexDigest`) are
  now wrapped with `StartNew[T]` from the package's Operations infrastructure.
  Each call gains automatic timing, structured logging, and panic recovery.
  `Digest.SumMD5` uses the operation name `"md5.Digest.SumMD5"` to distinguish
  it from the package-level `SumMD5`.

## [0.1.0] - 2026-03-23

### Added

- Initial package scaffolding generated by scaffold-generator

## [0.2.0] - 2026-03-23

### Added

- Full MD5 implementation following RFC 1321, written from scratch without
  using Go's `crypto/md5` standard library package.
- `SumMD5(data []byte) [16]byte` — one-shot API returning a fixed 16-byte array.
- `HexString(data []byte) string` — one-shot API returning a 32-char lowercase
  hex string.
- `Digest` struct — streaming hasher implementing `io.Writer` via `Write`.
- `New() *Digest` — constructor returning a freshly initialized `Digest`.
- `(*Digest).Write(p []byte) (int, error)` — feed data incrementally.
- `(*Digest).SumMD5() [16]byte` — non-destructive snapshot of the digest.
- `(*Digest).HexDigest() string` — non-destructive hex snapshot.
- 40 tests covering RFC 1321 vectors, output format, block boundaries (55, 56,
  64, 127, 128 bytes), edge cases, and all streaming scenarios.
- 100% statement coverage.
- Literate programming comments: T-table derivation, little-endian rationale,
  auxiliary function truth tables, Davies-Meyer feed-forward explanation,
  padding worked examples.
