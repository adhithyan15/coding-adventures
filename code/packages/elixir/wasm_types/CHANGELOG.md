## [0.2.0] - 2026-03-23

### Added

- Full WASM 1.0 type system implementation in `lib/coding_adventures/wasm_types.ex`
- `CodingAdventures.WasmTypes` module with:
  - `value_type/1` — returns binary byte tag for `:i32`, `:i64`, `:f32`, `:f64`
  - `block_type_empty/0` — returns 0x40 (empty block type tag)
  - `external_kind/1` — returns binary byte tag for `:function`, `:table`, `:memory`, `:global`
- Struct modules for all WASM 1.0 types:
  - `FuncType` — function signature (`params`, `results`)
  - `Limits` — min/max size constraints (`min`, `max`)
  - `MemoryType` — linear memory declaration (`limits`)
  - `TableType` — function reference table (`element_type` defaults to 0x70, `limits`)
  - `GlobalType` — global variable type + mutability (`value_type`, `mutable`)
  - `Import` — import declaration (`module_name`, `name`, `kind`, `type_info`)
  - `Export` — export declaration (`name`, `kind`, `index`)
  - `Global` — module-defined global with init expression (`global_type`, `init_expr`)
  - `Element` — table initialization segment (`table_index`, `offset_expr`, `function_indices`)
  - `DataSegment` — memory initialization segment (`memory_index`, `offset_expr`, `data`)
  - `FunctionBody` — function locals and bytecode (`locals`, `code`)
  - `CustomSection` — named tool metadata (`name`, `data`)
  - `WasmModule` — top-level module container (all fields default to empty/nil)
- 35 unit tests covering all types, constants, construction, field access, and edge cases
- Literate programming style: `@moduledoc` and `@doc` with ASCII diagrams, binary
  encoding tables, and explanations of WASM execution semantics
