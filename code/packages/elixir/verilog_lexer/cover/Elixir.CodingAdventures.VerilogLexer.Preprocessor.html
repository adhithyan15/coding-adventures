3 Verilog Preprocessor — Handles compiler directives before tokenization.
4
5 Verilog's preprocessor is modeled after C's preprocessor. Source files
6 can contain directives (lines starting with a backtick `` ` ``) that
7 control macro expansion, conditional compilation, and file inclusion.
8 These must be resolved before the lexer sees the source.
9
10 ## Why Preprocess Before Lexing?
11
12 Consider this Verilog source:
13
14 `define WIDTH 8
15 wire [`WIDTH-1:0] bus;
16
17 Without preprocessing, the lexer would see `` `WIDTH `` as a DIRECTIVE
18 token — it doesn't know that `WIDTH was defined as `8`. After
19 preprocessing, the lexer sees:
20
21 wire [8-1:0] bus;
22
23 Now `8`, `-`, `1`, `:`, and `0` are all normal tokens.
24
25 ## Supported Directives
26
27 ### `define / `undef — Macro Definition
28
29 `define WIDTH 8 — simple macro (text substitution)
30 `define MAX(a, b) (a > b ? a : b) — parameterized macro
31 `undef WIDTH — remove a macro definition
32
33 Simple macros replace `` `NAME `` with the defined text wherever it
34 appears. Parameterized macros accept arguments:
35
36 `MAX(x, y) → (x > y ? x : y)
37
38 ### `ifdef / `ifndef / `else / `endif — Conditional Compilation
39
40 `ifdef DEBUG
41 // This code is included only if DEBUG is defined
42 `else
43 // This code is included only if DEBUG is NOT defined
44 `endif
45
46 `ifndef is the inverse of `ifdef: code is included if the macro is
47 NOT defined. Conditionals can be nested.
48
49 ### `include — File Inclusion (Stubbed)
50
51 `include "definitions.vh"
52
53 In a real Verilog tool, this inserts the contents of the named file.
54 Our implementation replaces it with a comment placeholder:
55
56 // [include: definitions.vh]
57
58 This is stubbed because file I/O depends on the project structure and
59 search paths, which are beyond the scope of a standalone lexer.
60
61 ### `timescale — Time Unit Specification
62
63 `timescale 1ns/1ps
64
65 This directive sets the simulation time unit and precision. It has no
66 effect on the synthesizable design, so we strip it entirely (replace
67 with an empty string).
68
69 ## Implementation
70
71 The preprocessor works line-by-line:
72
73 1. Split the source into lines.
74 2. For each line, check if it starts with a directive (after stripping
75 leading whitespace).
76 3. Process the directive (define, undef, ifdef, etc.) and update state.
77 4. For non-directive lines, expand any macro references (`` `NAME ``).
78 5. Conditionally include or exclude lines based on `ifdef/`ifndef state.
79 6. Join the processed lines back together.
80
81 State is carried in a map with three fields:
82 - `:macros` — map of macro name to `{params, body}` tuples
83 - `:condition_stack` — stack of booleans tracking nested ifdef/ifndef
84 - `:active` — whether the current line should be included in output
85 """
