# ir-to-jvm-class-file

Prototype Python backend that lowers `compiler_ir.IrProgram` programs into real
JVM `.class` bytes.

## What it does

This package takes the repository's low-level AOT IR and emits a single JVM
class with:

- a static integer register file
- a static byte-array memory image
- one generated static JVM method per callable IR label
- helper methods for byte and word memory operations
- an optional `main(String[] args)` wrapper suitable for `java` and GraalVM

The design is intentionally conservative so that the generated bytecode is easy
to reason about and friendly to GraalVM Native Image.

## Example

```python
from compiler_ir import IrImmediate, IrInstruction, IrLabel, IrOp, IrProgram, IrRegister
from ir_to_jvm_class_file import JvmBackendConfig, lower_ir_to_jvm_class_file

program = IrProgram(entry_label="_start")
program.add_instruction(IrInstruction(IrOp.LABEL, [IrLabel("_start")], id=-1))
program.add_instruction(IrInstruction(IrOp.LOAD_IMM, [IrRegister(1), IrImmediate(0)], id=0))
program.add_instruction(IrInstruction(IrOp.HALT, [], id=1))

artifact = lower_ir_to_jvm_class_file(
    program,
    JvmBackendConfig(class_name="ExampleProgram"),
)

with open("ExampleProgram.class", "wb") as f:
    f.write(artifact.class_bytes)
```

## Frontend Integration

This package is generic over `IrProgram`, but the tests exercise it with real
frontends:

- Brainfuck source -> `brainfuck-ir-compiler` -> `IrProgram` -> `.class`
- Nib source -> `nib-ir-compiler` -> `IrProgram` -> `.class`

On this Mac, the test suite also validates two GraalVM execution paths:

- Brainfuck `.class` -> `java` -> expected stdout byte
- Brainfuck/Nib `.class` -> `native-image` -> native executable that runs locally

## Current Prototype Boundaries

- Class files are emitted directly; no Java source is generated.
- The backend uses a single shared register array instead of a full stack-frame
  model.
- Brainfuck uses the backend's syscall shim directly; Nib uses a tiny Java
  driver in tests to print the lowered program's `_start()` return value.
- The backend is sufficient for straight-line and branch-heavy static programs,
  and for the current Brainfuck and Nib paths.
