## [0.1.1] - 2026-03-23

### Added

- Full implementation of the WASM 1.0 type system:
  - `ValueType` IntEnum: I32=0x7F, I64=0x7E, F32=0x7D, F64=0x7C
  - `ExternalKind` IntEnum: FUNCTION=0x00, TABLE=0x01, MEMORY=0x02, GLOBAL=0x03
  - `BlockType` IntEnum: EMPTY=0x40
  - `FuncType` frozen dataclass: params/results as ValueType tuples
  - `Limits` frozen dataclass: min (required), max (optional)
  - `MemoryType` frozen dataclass: wraps Limits
  - `TableType` frozen dataclass: element_type (default 0x70=funcref) + Limits
  - `GlobalType` frozen dataclass: value_type + mutable flag
  - `Import` frozen dataclass: module_name, name, kind, type_info
  - `Export` frozen dataclass: name, kind, index
  - `Global` frozen dataclass: GlobalType + init_expr bytes
  - `Element` frozen dataclass: table_index, offset_expr, function_indices tuple
  - `DataSegment` frozen dataclass: memory_index, offset_expr, data bytes
  - `FunctionBody` frozen dataclass: locals tuple, code bytes
  - `CustomSection` frozen dataclass: name, data bytes
  - `WasmModule` mutable dataclass: all 12 section fields as lists
- 70 unit tests covering all 21 spec test scenarios; 100% line coverage
- Literate programming style: ASCII diagrams, binary encoding tables, examples
- `BUILD_windows` file for cross-platform CI
