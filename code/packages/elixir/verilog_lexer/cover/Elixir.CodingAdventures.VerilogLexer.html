9 ## What is Verilog?
10
11 Verilog is a Hardware Description Language (HDL) used to describe digital
12 circuits. Unlike software languages that describe sequential computations,
13 Verilog describes physical structures — gates, wires, flip-flops — that
14 exist simultaneously and operate in parallel.
15
16 ## Preprocessor
17
18 Verilog source files often contain compiler directives (lines starting with
19 a backtick `` ` ``). These directives control conditional compilation, macro
20 expansion, and file inclusion — similar to the C preprocessor. The
21 `VerilogLexer.Preprocessor` module handles these before the source reaches
22 the lexer.
23
24 Supported directives:
25 - `` `define `` / `` `undef `` — macro definition and removal
26 - `` `ifdef `` / `` `ifndef `` / `` `else `` / `` `endif `` — conditional compilation
27 - `` `include `` — file inclusion (stubbed: emits a comment placeholder)
28 - `` `timescale `` — time unit specification (stripped entirely)
29 - Parameterized macros: `` `define ADD(a, b) (a + b) ``
30
31 ## Usage
32
33 # Basic tokenization (no preprocessing)
34 {:ok, tokens} = CodingAdventures.VerilogLexer.tokenize("module foo; endmodule")
35
36 # With preprocessing enabled
37 source = ~s(`define WIDTH 8\\nwire [`WIDTH-1:0] bus;)
38 {:ok, tokens} = CodingAdventures.VerilogLexer.tokenize(source, preprocess: true)
39
40 ## How It Works
41
42 1. If `preprocess: true` is passed, the source is first run through
43 `VerilogLexer.Preprocessor.process/1` to expand macros, evaluate
44 conditionals, and strip directives.
45
46 2. The (possibly preprocessed) source is passed to `GrammarLexer.tokenize/2`
