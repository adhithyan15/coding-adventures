# Changelog

All notable changes to this package will be documented in this file.

## [0.2.0] - 2026-03-31

### Changed

- Wrapped public functions `GetOpcode` and `GetOpcodeByName` with the Operations system (`StartNew[T]`), providing automatic timing, structured logging, and panic recovery. Public API signatures unchanged.

## [0.1.0] - 2026-03-23

### Added

- Initial package scaffolding generated by scaffold-generator

## [0.1.1] - 2026-03-23

### Added

- Full implementation of the `wasm-opcodes` Go package (`wasmopcodes`)
- `OpcodeInfo` struct with fields: `Name`, `Opcode`, `Category`,
  `Immediates`, `StackPop`, `StackPush`
- `Opcodes map[byte]OpcodeInfo` — lookup by byte value (package-level var)
- `OpcodesByName map[string]OpcodeInfo` — lookup by mnemonic (package-level var)
- `GetOpcode(byte) (OpcodeInfo, bool)` — None-safe lookup by byte
- `GetOpcodeByName(string) (OpcodeInfo, bool)` — None-safe lookup by name
- Complete table of all 172 WASM 1.0 instructions across 9 categories:
  control, parametric, variable, memory (loads, stores, management),
  numeric_i32, numeric_i64, numeric_f32, numeric_f64, conversion
- Both maps built at `init()` time from a single `rawTable` source of truth
- 49 unit tests with 100% statement coverage
- Literate programming: inline explanations of structured control flow,
  memarg encoding, stack effects, and conversion naming conventions
