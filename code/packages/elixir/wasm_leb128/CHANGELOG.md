## [0.2.0] - 2026-03-23

### Added

- Full implementation of LEB128 unsigned and signed encoding/decoding
- `decode_unsigned(data, offset \\ 0)` — decodes an unsigned LEB128 integer
  from a binary using Elixir binary pattern matching; returns
  `{:ok, {value, bytes_consumed}}` or `{:error, message}`
- `decode_signed(data, offset \\ 0)` — decodes a signed LEB128 integer with
  sign-extension using Elixir's arbitrary-precision integers; returns
  `{:ok, {value, bytes_consumed}}` or `{:error, message}`
- `encode_unsigned(value)` — encodes a non-negative integer as a LEB128 binary
- `encode_signed(value)` — encodes a signed integer as LEB128 using two's
  complement with proper termination condition
- 42 unit tests plus 12 doctests covering all required test cases
- Literate programming style: all functions documented with algorithm
  walkthroughs, visual traces, and examples

### Implementation Notes

- Uses `Integer.floor_div/2` for arithmetic right shift in `encode_signed`,
  since `Bitwise.asr/2` does not exist in Elixir 1.14+. Floor division by
  128 (= 2^7) gives identical semantics for negative numbers.
- Sign extension in `decode_signed` uses Elixir's infinite-precision integers:
  `-(1 <<< shift)` produces a mask with all bits above `shift` set to 1, then
  we mask to 64 bits and convert to a signed 64-bit value.
- The commonly cited example `<<0xE5, 0x88, 0x26>>` for 624485 is incorrect;
  the correct encoding is `<<0xE5, 0x8E, 0x26>>` (0x88 decodes to 623717).
