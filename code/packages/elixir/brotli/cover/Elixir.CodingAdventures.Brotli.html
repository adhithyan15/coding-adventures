5 Brotli Compression — CMP06 wire-format compress/decompress.
6
7 ## What Is Brotli?
8
9 Brotli (2013, RFC 7932) is a lossless compression algorithm developed at Google that
10 achieves significantly better compression ratios than DEFLATE, particularly on web
11 content (HTML, CSS, JavaScript). It became the dominant algorithm for HTTP
12 `Content-Encoding: br` compression.
13
14 Brotli builds on DEFLATE's foundation but adds three major innovations:
15
16 1. **Context-dependent literal trees** — instead of one Huffman tree for all literals,
17 Brotli assigns each literal to one of 4 *context buckets* based on the preceding
18 byte. Each context bucket gets its own Huffman tree.
19
20 2. **Insert-and-copy commands** — instead of separate literal and back-reference tokens,
21 Brotli uses *commands* that bundle an insert run (raw literals) with a copy
22 operation. Both lengths are encoded together as a single ICC Huffman symbol.
23
24 3. **Larger sliding window** — 65535 bytes instead of DEFLATE's 4096.
25
26 ## Encoding Order
27
28 Each regular command (copy_length >= 4) in the bit stream:
29
30 ```
31 [ICC Huffman code]
32 [insert_length extra bits, LSB-first]
33 [copy_length extra bits, LSB-first]
34 [literal bytes, each encoded via per-context Huffman tree]
35 [distance code Huffman code]
36 [distance extra bits, LSB-first]
37 ```
38
39 End of stream:
40
41 ```
42 [ICC=63 sentinel]
43 [flush literal bytes, if any — encoded AFTER the sentinel]
44 ```
45
46 ## Context Buckets
47
48 Each literal is assigned to a context based on the preceding byte:
49
50 ```
51 bucket 0 — space or punctuation (or start-of-stream: no previous byte)
52 bucket 1 — digit ('0'–'9')
53 bucket 2 — uppercase letter ('A'–'Z')
54 bucket 3 — lowercase letter ('a'–'z')
55 ```
56
57 ## Flush Literals
58
59 Trailing literals that cannot be bundled into a regular ICC command (because
60 there is no LZ match after them) are encoded **after** the sentinel ICC=63.
61 The decoder reads them after breaking out of the ICC loop, until `original_length`
62 bytes have been produced.
63
64 ## Wire Format (CMP06)
65
66 ```
67 Header (10 bytes):
68 [4B] original_length big-endian uint32
69 [1B] icc_entry_count entries in ICC table (1-64)
70 [1B] dist_entry_count entries in dist table (0-32)
71 [1B] ctx0_entry_count entries in literal tree 0
72 [1B] ctx1_entry_count entries in literal tree 1
73 [1B] ctx2_entry_count entries in literal tree 2
74 [1B] ctx3_entry_count entries in literal tree 3
75
76 ICC table (icc_entry_count × 2 bytes): symbol::8, code_length::8
77 Dist table (dist_entry_count × 2 bytes): symbol::8, code_length::8
78 Literal trees 0-3 (entry_count × 3 bytes each): symbol::16, code_length::8
79 Bit stream: LSB-first packed bits, zero-padded to byte boundary.
80 ```
81
82 ## Examples
83
