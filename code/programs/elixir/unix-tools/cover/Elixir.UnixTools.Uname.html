3 uname -- print system information.
4
5 ## What This Program Does
6
7 This is a reimplementation of the GNU `uname` utility in Elixir. It prints
8 various pieces of system information depending on which flags are provided.
9
10 ## How uname Works
11
12 With no flags, `uname` prints just the kernel name:
13
14 uname => Darwin (on macOS)
15 uname => Linux (on Linux)
16
17 With `-a` (all), it prints everything:
18
19 uname -a => Darwin hostname 23.1.0 Darwin Kernel Version 23.1.0 x86_64
20
21 ## Available Information Fields
22
23 | Flag | Field | Source in Erlang/OTP |
24 |------|------------------|-----------------------------------------|
25 | -s | Kernel name | `:os.type()` -> {family, _} |
26 | -n | Node name | `:inet.gethostname()` |
27 | -r | Kernel release | `:os.cmd(~c"uname -r")` |
28 | -v | Kernel version | `:os.cmd(~c"uname -v")` |
29 | -m | Machine | `:erlang.system_info(:system_architecture)` |
30 | -p | Processor | Same as machine on most systems |
31 | -i | Hardware platform| Same as machine on most systems |
32 | -o | Operating system | Derived from `:os.type()` |
33
34 ## Implementation Notes
35
36 We use Erlang/OTP's built-in system information functions where possible.
37 For fields not directly available (kernel release, kernel version), we
38 fall back to calling the system `uname` command. This is pragmatic --
39 these values are OS-specific and not abstracted by the BEAM.
40 """

106 @doc """
107 Build the output string from system info and selected fields.
108
109 ## How Field Selection Works
110
111 If no specific field flags are set, only `:kernel_name` is shown.
112 If `-a` is set, all fields are shown in a fixed order.
113 Otherwise, only the explicitly requested fields are shown.
114
115 The fields are printed space-separated, in the canonical order:
116 kernel_name, nodename, kernel_release, kernel_version, machine,
117 processor, hardware_platform, operating_system.
118
119 ## Examples
120
121 iex> info = %{kernel_name: "Linux", nodename: "box", ...}
122 iex> UnixTools.Uname.format_output(info, %{all: true})
123 "Linux box 5.4.0 #1 SMP x86_64 x86_64 x86_64 GNU/Linux"
124 """

146

:-(

150

:-(

153 field_order

:-(

162 
:-(
