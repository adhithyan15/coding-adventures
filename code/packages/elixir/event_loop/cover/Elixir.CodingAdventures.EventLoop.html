5 ## What is an event loop?
6
7 An event loop is the outermost structure of any interactive program. It runs
8 forever (until told to stop), repeatedly asking "did anything happen?" and
9 dispatching whatever happened to registered handlers:
10
11 while running:
12 collect events from all sources
13 for each event:
14 dispatch to handlers
15 if any handler says :exit → stop
16
17 ## Why functional in Elixir?
18
19 Unlike Go or Rust where the loop mutates struct fields, Elixir uses
20 immutable values and tail recursion. The loop is a recursive function
21 that passes updated source state forward on each iteration — no mutation,
22 no side effects except in the handlers you provide.
23
24 This models a fundamental Elixir pattern: evolving state through function
25 calls rather than mutation.
26
27 ## Sources in Elixir
28
29 A source is a `{poll_fn, state}` tuple where:
30 - `poll_fn` is a function of arity 1: `fn state -> {[events], new_state}`
31 - `state` is any term (the source's current state)
32
33 On each iteration the loop calls `poll_fn.(state)` and uses `new_state`
34 for the next iteration. This is the standard Elixir way to handle stateful
35 computation without mutation.
36
37 ## Handlers
38
39 A handler is a function of arity 1: `fn event -> :continue | :exit`.
40
