3 Sequential Logic — memory elements built from logic gates.
4
5 ## From Gates to Memory
6
7 Combinational gates (AND, OR, NOT) have no memory — their output depends
8 only on their current inputs. But computers need to remember things:
9 register values, program counters, cached data.
10
11 Sequential logic creates memory by feeding a gate's output back into
12 its own input. This creates a feedback loop that can "hold" a value
13 even after the input changes. The simplest example is the SR latch:
14 two NOR gates cross-coupled so each gate's output feeds into the
15 other gate's input.
16
17 ## The Memory Hierarchy Built from Latches
18
19 SR Latch → simplest memory (1 bit, set/reset)
20 │
21 D Latch → controlled memory (1 bit, data + enable)
22 │
23 D Flip-Flop → edge-triggered (1 bit, data + clock)
24 │
25 Register → N-bit word storage (parallel flip-flops)
26 │
27 Shift Register → serial-to-parallel conversion
28 │
29 Counter → binary counting
30
31 Each level builds on the previous one. A CPU register file is just
32 a collection of registers. A register is just a row of flip-flops.
33 A flip-flop is just two latches. A latch is just two cross-coupled
34 NOR gates. It's gates all the way down.

46 An SR latch is built from two NOR gates with cross-coupled feedback:

48
49 Set ────┐
50 NOR ──── Q (output)
51 ┌───┘ └───┐
52 │ │ (cross-coupled feedback)
53 └───┐ ┌───┘
