# X25519 — Elliptic Curve Diffie-Hellman Key Exchange

## Overview

X25519 (RFC 7748) is an elliptic curve Diffie-Hellman (ECDH) function
on Curve25519. Designed by Daniel J. Bernstein in 2006, it computes a
shared secret between two parties using only x-coordinates on a
Montgomery curve — no y-coordinate needed, no point compression, no
special cases.

Used in: TLS 1.3, SSH, WireGuard, Signal Protocol, Noise Framework,
libsodium, and nearly every modern key exchange deployment.

### Why X25519?

- **Constant-time by design:** The Montgomery ladder processes every
  bit of the scalar identically — no branches, no timing leaks. You
  have to go out of your way to implement it *wrong*.
- **Simple:** One function, two 32-byte inputs, one 32-byte output.
  No point validation needed (all 32-byte strings are valid inputs).
- **Fast:** ~150,000 operations/second on a modern CPU.
- **Misuse-resistant:** Key clamping prevents small-subgroup attacks
  and ensures the result is always in the prime-order subgroup.

### The Curve

X25519 operates on the Montgomery curve:

```
y^2 = x^3 + 486662 * x^2 + x    (mod p)

where p = 2^255 - 19
```

This is the same curve as Ed25519, just in Montgomery form instead of
twisted Edwards form. The constant 486662 is the smallest value
producing a curve with the desired security properties (prime-order
subgroup of size L, cofactor 8).

### Base Point

The base point has u-coordinate (x-coordinate in Montgomery form):

```
u(B) = 9
```

That's it. Just 9. The v-coordinate (y-coordinate) is not needed for
X25519 — this is the beauty of the Montgomery ladder.

## Algorithm

### Scalar Clamping

Before any scalar multiplication, the 32-byte scalar is "clamped":

```
Clamp(k):
    k[0]  &= 248     // Clear the lowest 3 bits
    k[31] &= 127     // Clear the highest bit
    k[31] |= 64      // Set the second-highest bit
```

Why each step:
- **Clear low 3 bits:** The curve has cofactor 8 = 2^3. Clearing
  these bits ensures the scalar is a multiple of 8, so the result is
  always in the prime-order subgroup. This prevents small-subgroup
  attacks without requiring point validation.
- **Clear high bit:** Ensures the scalar fits in 255 bits (since
  p = 2^255 - 19, we work in at most 255 bits).
- **Set second-highest bit:** Makes all scalars have the same bit
  length, ensuring the Montgomery ladder always runs exactly 255
  iterations. This provides constant-time execution regardless of
  the scalar value.

### Montgomery Ladder

The core of X25519 is scalar multiplication using only x-coordinates
(called u-coordinates in RFC 7748). Given scalar k and u-coordinate u,
compute k * (u, ?) and return only the resulting u-coordinate.

```
x25519(k, u):
    k = Clamp(k)
    u = u mod p                          // Decode u as little-endian, reduce mod p
                                         // (Note: high bit of u is masked to 0)

    // Montgomery ladder
    x_1 = u
    x_2 = 1;    z_2 = 0                 // Projective coords: point at infinity
    x_3 = u;    z_3 = 1                 // Projective coords: base point

    swap = 0
    for i = 254 down to 0:
        k_i = (k >> i) & 1
        swap ^= k_i                     // Conditional swap
        (x_2, x_3) = cswap(swap, x_2, x_3)
        (z_2, z_3) = cswap(swap, z_2, z_3)
        swap = k_i

        A  = x_2 + z_2
        AA = A^2
        B  = x_2 - z_2
        BB = B^2
        E  = AA - BB
        C  = x_3 + z_3
        D  = x_3 - z_3
        DA = D * A
        CB = C * B
        x_3 = (DA + CB)^2
        z_3 = x_1 * (DA - CB)^2
        x_2 = AA * BB
        z_2 = E * (AA + a24 * E)        // a24 = 121666 = (486662 + 2) / 4

    (x_2, x_3) = cswap(swap, x_2, x_3)
    (z_2, z_3) = cswap(swap, z_2, z_3)

    Return x_2 * z_2^(p-2) mod p        // Convert from projective: x/z
    Encode as 32 bytes little-endian
```

All arithmetic is mod p = 2^255 - 19.

### Constant-Time Conditional Swap (cswap)

```
cswap(swap, a, b):
    // If swap=1, exchange a and b. If swap=0, do nothing.
    // Must be constant-time (no branches).
    mask = 0 - swap                      // 0x0000...0 or 0xFFFF...F
    t = mask & (a ^ b)
    a ^= t
    b ^= t
    return (a, b)
```

### The Constant a24

The Montgomery ladder uses `a24 = (A + 2) / 4` where A = 486662:

```
a24 = (486662 + 2) / 4 = 121666
```

This constant appears in the inner loop and is the only curve-specific
value needed.

### Diffie-Hellman Key Exchange

```
Alice:                              Bob:
  a = random 32 bytes                b = random 32 bytes
  A = x25519(a, 9)                   B = x25519(b, 9)

  // Exchange A and B over public channel

  shared = x25519(a, B)             shared = x25519(b, A)

  // shared == shared (same value!)
  // Because: a * (b * G) == b * (a * G)
```

Both parties arrive at the same shared secret, which is then typically
fed into a KDF (like HKDF) to derive encryption keys.

**Important:** If the result is the all-zeros value (32 zero bytes),
the implementation MUST abort — this indicates the peer sent a
point in the small subgroup (a malicious or broken peer).

## Interface Contract

| Function | Signature | Description |
|----------|-----------|-------------|
| `x25519` | `(scalar_32, u_coordinate_32) -> u_coordinate_32` | Scalar multiplication on Curve25519. |
| `x25519_base` | `(scalar_32) -> u_coordinate_32` | Scalar multiplication by base point (u=9). Equivalent to `x25519(scalar, 9)`. |
| `generate_keypair` | `(private_key_32) -> public_key_32` | Derive public key from private key. Same as `x25519_base`. |

All inputs and outputs are 32 bytes, little-endian encoded.

Constraints:
- Input u-coordinate: high bit (bit 255) is masked to 0 before use.
- Input scalar: clamped before use (low 3 bits cleared, high bit
  cleared, second-highest bit set).
- Output: if the result is 32 zero bytes, reject (abort/error).
