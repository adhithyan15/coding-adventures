# mosaic-emit-webcomponent

Web Components backend for the Mosaic compiler pipeline. Takes a `MosaicIR`
component description and emits a `.ts` file containing a `HTMLElement` subclass
registered as a custom element with the browser's `customElements` registry.

## Where it fits in the pipeline

```
.mosaic source
    в””в”Ђв–є mosaic-lexer  (tokens)
    в””в”Ђв–є mosaic-parser  (ASTNode tree)
    в””в”Ђв–є mosaic-analyzer  (MosaicIR вЂ” typed IR)
    в””в”Ђв–є mosaic-vm  (drives traversal, calls renderer)
    в””в”Ђв–є mosaic-emit-webcomponent  в†ђ YOU ARE HERE
              в””в”Ђв–є mosaic-profile-card.ts
```

## Usage

```typescript
import { MosaicWebComponentRenderer } from "@coding-adventures/mosaic-emit-webcomponent";
import { MosaicVM } from "@coding-adventures/mosaic-vm";
import { analyzeMosaic } from "@coding-adventures/mosaic-analyzer";

const ir = analyzeMosaic(source);          // parse + analyze .mosaic source
const vm = new MosaicVM(ir);
const result = vm.run(new MosaicWebComponentRenderer());

// result.files[0].filename  в†’ "mosaic-profile-card.ts"
// result.files[0].content   в†’ "// AUTO-GENERATED ..."
```

## What gets generated

Given a Mosaic component like:

```mosaic
component ProfileCard {
  slot name: text
  slot age: number

  Box {
    padding: 16dp;
    Text {
      content: @name;
    }
  }
}
```

The renderer produces a `mosaic-profile-card.ts` file containing a complete
Custom Element class:

```typescript
// AUTO-GENERATED by mosaic-emit-webcomponent вЂ” do not edit by hand

class MosaicProfileCardElement extends HTMLElement {
  private _name: string = '';
  private _age: number = 0;

  static get observedAttributes() {
    return ['name', 'age'];
  }

  attributeChangedCallback(name: string, _old: string | null, value: string | null) {
    if (name === 'name') { this.name = value ?? ''; }
    if (name === 'age') { this.age = Number(value ?? 0); }
  }

  get name(): string { return this._name; }
  set name(v: string) { this._name = v; this._render(); }

  get age(): number { return this._age; }
  set age(v: number) { this._age = v; this._render(); }

  connectedCallback() { this.attachShadow({ mode: 'open' }); this._render(); }

  private _escapeHtml(s: string): string {
    return String(s)
      .replace(/&/g, '&amp;')
      .replace(/</g, '&lt;')
      .replace(/>/g, '&gt;')
      .replace(/"/g, '&quot;');
  }

  private _render() {
    if (!this.shadowRoot) return;
    let html = '';
    html += '<div style="display:flex;padding:16px;">';
    html += '<span>';
    html += this._escapeHtml(String(this._name));
    html += '</span>';
    html += '</div>';
    this.shadowRoot.innerHTML = html;
  }
}

customElements.define('mosaic-profile-card', MosaicProfileCardElement);
```

## Output conventions

| Mosaic concept | Web Component output |
|---|---|
| `Box` | `<div>` |
| `Text` | `<span>` |
| `Image` | `<img>` |
| `Button` | `<button>` |
| Component name `ProfileCard` | tag `mosaic-profile-card`, class `MosaicProfileCardElement` |
| `slot foo: text` | `private _foo: string = ''` + getter/setter + observedAttributes |
| `slot bar: number` | `private _bar: number = 0` + getter/setter |
| `slot baz: bool` | `private _baz: boolean = false` + getter/setter |
| `slot xs: list<text>` | `private _xs: string[] = []` + getter/setter |
| `slot qux: node` | Light DOM projection via `<slot name="qux">` element |
| `@slotName` in property | `this._escapeHtml(String(this._slotName))` |
| `when @flag { ... }` | `if (this._flag) { html += ...; }` |
| `each @items as item { ... }` | `this._items.forEach((item, _index) => { ... })` |
| `a11y-role: button` | `role="button"` HTML attribute |
| `a11y-hidden: true` | `aria-hidden="true"` HTML attribute |
| `color: #rrggbbaa` | `rgba(r,g,b,alpha)` in inline style |
| `padding: 16dp` | `padding:16px` in inline style string |
| CSS properties | kebab-case in inline `style="..."` |

The custom element uses **Shadow DOM** (`attachShadow({ mode: 'open' })`) for
encapsulation. `node`-typed slots use Light DOM projection (the `<slot>` element
is inserted into the shadow tree, and consumers place children in the Light DOM).

String content is always escaped with `_escapeHtml()` to prevent XSS. Image
`src` values generated from slot references validate against `javascript:` URLs.
