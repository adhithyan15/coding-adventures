3 diff -- compare files line by line.
4
5 ## What This Program Does
6
7 This is a reimplementation of the GNU `diff` utility in Elixir. It compares
8 two files (or directories) and reports the differences between them.
9
10 ## How diff Works
11
12 At its simplest:
13
14 diff file1.txt file2.txt => shows differences between the two files
15
16 ## Output Formats
17
18 diff supports several output formats:
19
20 | Format | Flag | Description |
21 |-----------|------|----------------------------------------------------|
22 | Normal | - | Default: shows change commands (a, d, c) |
23 | Unified | -u | Shows context with +/- prefixes (like git diff) |
24 | Context | -c | Shows context with ! markers |
25
26 ## The LCS Algorithm
27
28 The heart of diff is the **Longest Common Subsequence (LCS)** algorithm.
29 Given two sequences, LCS finds the longest sequence of elements that appear
30 in both, in the same order (but not necessarily contiguous).
31
32 For example:
33 File 1: A B C D E
34 File 2: A C D F E
35
36 LCS: A C D E (length 4)
37
38 The lines NOT in the LCS are the differences:
39 - B was deleted (in file1 but not LCS)
40 - F was inserted (in file2 but not LCS)
41
42 ## How LCS Works (Dynamic Programming)
43
44 We build a 2D table where `dp[i][j]` = length of LCS of first i lines of
45 file1 and first j lines of file2.
46
47 The recurrence:
48 - If `lines1[i] == lines2[j]`: `dp[i][j] = dp[i-1][j-1] + 1`
49 - Otherwise: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`
50
51 Then we backtrack through the table to find the actual LCS.
52
53 ## Normalization Flags
54
55 - `-i` (ignore-case): Compare lines case-insensitively.
56 - `-b` (ignore-space-change): Collapse runs of whitespace to a single space.
57 - `-w` (ignore-all-space): Remove all whitespace before comparing.
58 - `-B` (ignore-blank-lines): Skip blank lines entirely.
59
60 ## Implementation Approach
61
62 1. `compute_lcs/3` finds the longest common subsequence using DP.
63 2. `compute_edit_script/3` converts LCS into an edit script (add/delete/change).
64 3. `format_normal/3`, `format_unified/4`, `format_context/4` render the output.
65 """
