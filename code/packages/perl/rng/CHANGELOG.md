# Changelog

All notable changes to this package will be documented in this file.

## [0.01] - 2026-04-25

### Added

- Initial package scaffolding generated by scaffold-generator

### Implemented

- `CodingAdventures::Rng::LCG`: Linear Congruential Generator (Knuth 1948).
  Constants: multiplier = 6364136223846793005, increment = 1442695040888963407.
  Uses Math::BigInt for 64-bit multiply. Output is upper 32 bits of state.
- `CodingAdventures::Rng::Xorshift64`: Marsaglia (2003) XOR-shift generator.
  Three shifts (13, 7, 17), period 2^64 − 1. Seed 0 → 1. Output is lower 32 bits.
  Fixed a subtle Math::BigInt in-place mutation bug: state was being corrupted
  when the return value mask (u32) was applied after saving state because both
  variables pointed to the same BigInt object. Fixed by using `.copy()` and
  saving state before extracting the return value.
- `CodingAdventures::Rng::PCG32`: O'Neill (2014) Permuted Congruential Generator.
  LCG recurrence plus XSH RR output permutation. Two-step initseq warm-up.
- All three classes implement: `next_u32`, `next_u64`, `next_float`, `next_int_in_range`.
- `next_int_in_range` uses rejection sampling to eliminate modulo bias.
- `next_u64` returns a `Math::BigInt` object.
- Reference values match the Go implementation exactly for seed=1:
  - LCG:        [1817669548, 2187888307, 2784682393]
  - Xorshift64: [1082269761, 201397313,  1854285353]
  - PCG32:      [1412771199, 1791099446, 124312908]
- 59 tests across 5 test files (Test2::V0): load/version, API presence,
  reference values, determinism, output ranges, full range coverage, and
  single-value range check.
