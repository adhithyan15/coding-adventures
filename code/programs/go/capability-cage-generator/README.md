# capability-cage-generator

Reads `required_capabilities.json` at build time and emits `gen_capabilities.go` that bakes capability declarations into the binary as compiled-in constants.

## Why

If capabilities were declared in a JSON file read at runtime, an attacker with filesystem write access could escalate permissions by editing the JSON before the program reads it. By generating Go source instead, capabilities become compile-time constants — immutable once the binary is built.

## Usage

```bash
# Single package
capability-cage-generator --manifest=code/packages/go/verilog-lexer/required_capabilities.json

# All Go packages in the repo
capability-cage-generator --all

# Preview without writing
capability-cage-generator --manifest=... --dry-run
capability-cage-generator --all --dry-run
```

## What It Generates

Given `required_capabilities.json`:
```json
{
  "package": "go/verilog-lexer",
  "capabilities": [
    {
      "category": "fs",
      "action": "read",
      "target": "*",
      "justification": "Reads grammar files at startup to configure the lexer DFA."
    }
  ]
}
```

Produces `gen_capabilities.go`:
```go
// Code generated by capability-cage-generator. DO NOT EDIT.
package veriloglexer

import cage "github.com/adhithyan15/coding-adventures/code/packages/go/capability-cage"

// Manifest declares the OS-level capabilities of this package,
// compiled in at build time. No JSON file is read at runtime.
var Manifest = cage.NewManifest([]cage.Capability{
    {
        Category:      cage.CategoryFS,
        Action:        cage.ActionRead,
        Target:        "*",
        Justification: "Reads grammar files at startup to configure the lexer DFA.",
    },
})
```

For pure-computation packages (empty capabilities):
```go
var Manifest = cage.EmptyManifest
```

## Package Name Derivation

1. Read `package` declaration from any `.go` file in the same directory (most accurate).
2. Fallback: derive from the `package` field in JSON — take the last segment, remove hyphens.
   - `go/verilog-lexer` → `veriloglexer`

## Language Support

Only Go packages (where the `package` field starts with `go/`) are processed. Other languages are skipped with a message.
