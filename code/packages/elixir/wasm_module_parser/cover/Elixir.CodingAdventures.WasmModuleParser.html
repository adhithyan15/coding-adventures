10 ## The WebAssembly Binary Format
11
12 A `.wasm` file is a compact binary encoding of a WebAssembly module.
13 Every integer uses LEB128 variable-length encoding. Strings are
14 length-prefixed UTF-8.
15
16 ```
17 ┌──────────────────────────────────────────────────────────────────────────┐
18 │ WASM Binary Layout │
19 ├──────────────────────────────────────────────────────────────────────────┤
20 │ Magic │ Version │ Section │ Section │ ... │
21 │ 4 bytes │ 4 bytes │ id+size+payload │ │
22 └──────────────────────────────────────────────────────────────────────────┘
23
24 Magic: <<0x00, 0x61, 0x73, 0x6D>> (the bytes "\\0asm")
25 Version: <<0x01, 0x00, 0x00, 0x00>> (little-endian u32 = 1)
26
27 Each section:
28 ┌──────┬──────────────────┬────────────────────────────────────────┐
29 │ id │ size (u32 leb128)│ payload (size bytes) │
30 │ 1 B │ 1–5 bytes │ contents vary by id │
31 └──────┴──────────────────┴────────────────────────────────────────┘
32
33 Section IDs:
34 0 = Custom any position, any number of times
35 1 = Type function type signatures
36 2 = Import host-provided imports
37 3 = Function type index for each local function
38 4 = Table indirect-call tables
39 5 = Memory linear memory declarations
40 6 = Global module-level global variables
41 7 = Export names exported to the host
42 8 = Start optional auto-called function
43 9 = Element table initialisation data
44 10 = Code function bodies (locals + bytecode)
45 11 = Data memory initialisation data
46 ```
47
