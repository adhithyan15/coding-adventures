2 @moduledoc """
3 df -- report file system disk space usage.
4
5 ## What This Program Does
6
7 This is a reimplementation of the GNU `df` utility in Elixir. It displays
8 information about the file system on which each FILE resides, or all file
9 systems by default.
10
11 ## How df Works
12
13 With no arguments, `df` shows all mounted file systems:
14
15 df => Filesystem 1K-blocks Used Available Use% Mounted on
16 /dev/sda1 102400000 60000000 42400000 59% /
17 tmpfs 8192000 0 8192000 0% /tmp
18
19 With a path argument, it shows only the file system containing that path:
20
21 df /home => just the entry for /home's filesystem
22
23 ## Display Options
24
25 - `-h` (--human-readable): Show sizes in powers of 1024 (e.g., 1.5G).
26 - `-H` (--si): Show sizes in powers of 1000 (e.g., 1.6G).
27 - `-T` (--print-type): Include the filesystem type column.
28 - `-i` (--inodes): Show inode usage instead of block usage.
29
30 ## Implementation Approach
31
32 We delegate to the system `df` command via `:os.cmd/1` and parse its
33 tabular output. The heavy lifting is in parsing the column-based output
34 that `df` produces, which can vary between operating systems.
35
36 The business logic (parsing df output, formatting sizes) is kept as
37 pure functions for testability.
38 """
