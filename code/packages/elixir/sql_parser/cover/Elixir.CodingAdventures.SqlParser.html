9 ## Usage

12
13 The returned AST is a tree of `ASTNode` structs where `rule_name`

15 `"expr"`) and `children` contains sub-nodes and tokens.
16
17 ## Supported SQL Statements
18
19 The grammar covers an ANSI SQL subset:
20
21 - `SELECT … FROM … [WHERE …] [GROUP BY …] [HAVING …] [ORDER BY …] [LIMIT …]`
22 - `INSERT INTO … VALUES (…)`
23 - `UPDATE … SET …`
24 - `DELETE FROM …`
25 - `CREATE TABLE … (…)`
26 - `DROP TABLE …`
27
28 Multiple statements separated by `;` are all parsed under the root
29 `"program"` node.
30
31 ## Case-Insensitive Keywords
32
33 The SQL lexer normalizes all keyword values to uppercase (see `SqlLexer`).
34 The grammar matches quoted keyword literals like `"SELECT"` against the
35 already-uppercase token values, so keyword case in source is irrelevant.
36 """
