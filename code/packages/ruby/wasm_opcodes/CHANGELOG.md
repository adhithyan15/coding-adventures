## [0.1.1] - 2026-03-23

### Added

- Full implementation in `lib/coding_adventures/wasm_opcodes/opcodes.rb` with
  all 172 WASM 1.0 instructions
- `OpcodeInfo` Struct with fields: `name`, `opcode`, `category`, `immediates`,
  `stack_pop`, and `stack_push` (keyword_init: true)
- `OPCODES` — primary `Hash[Integer, OpcodeInfo]` keyed by opcode byte
- `OPCODES_BY_NAME` — secondary `Hash[String, OpcodeInfo]` keyed by name
- `WasmOpcodes.get_opcode(byte)` — look up by single-byte encoding
- `WasmOpcodes.get_opcode_by_name(name)` — look up by text-format name
- Comprehensive test suite (41 tests, 576 assertions) covering:
  - Table size and consistency
  - Byte and name lookups
  - Stack effect metadata (stack_pop / stack_push)
  - Immediate operand types (memarg, blocktype, funcidx, etc.)
  - Data integrity (unique bytes, unique names, non-empty names)
  - All 9 instruction categories
- Literate programming comments explaining the WASM stack machine model,
  immediate operand encoding formats, and instruction categories
- Fixed Gemfile typo: `coding_adventures_/wasm_leb128` → `coding_adventures_wasm_leb128`
- All 172 instructions across 9 categories:
  - `control` (13 instructions: unreachable through call_indirect)
  - `parametric` (2: drop, select)
  - `variable` (5: local.get/set/tee, global.get/set)
  - `memory` (25: 14 loads + 9 stores + memory.size + memory.grow)
  - `numeric_i32` (30: const, comparisons, arithmetic, bitwise)
  - `numeric_i64` (30: const, comparisons, arithmetic, bitwise)
  - `numeric_f32` (21: const, comparisons, arithmetic)
  - `numeric_f64` (21: const, comparisons, arithmetic)
  - `conversion` (25: wrap, extend, trunc, convert, demote, promote, reinterpret)
