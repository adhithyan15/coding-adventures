# Changelog

All notable changes to this package will be documented in this file.

## [0.1.0] - 2026-03-23

### Added

- Initial package scaffolding generated by scaffold-generator
- `ValueType` const object with all four WASM 1.0 numeric type codes:
  `I32` (0x7F), `I64` (0x7E), `F32` (0x7D), `F64` (0x7C)
- `BlockType` const with `EMPTY` (0x40) for empty-result blocks
- `ExternalKind` const object with `FUNCTION` (0x00), `TABLE` (0x01),
  `MEMORY` (0x02), `GLOBAL` (0x03)
- `FUNCREF` constant (0x70) for the table element type
- `FuncType` interface and `makeFuncType` factory — immutable function
  type signatures with `readonly` params and results arrays
- `Limits` interface — min/max page/element bounds for memories and tables
- `MemoryType` interface — linear memory type wrapping Limits
- `TableType` interface — funcref table type with elementType and Limits
- `GlobalType` interface — value type plus mutability flag
- `Import` interface — two-part name, ExternalKind, and kind-specific typeInfo
- `Export` interface — name, ExternalKind, and index into index space
- `Global` interface — GlobalType plus raw init-expression bytes
- `Element` interface — table initializer: tableIndex, offsetExpr, functionIndices
- `DataSegment` interface — memory initializer: memoryIndex, offsetExpr, data
- `FunctionBody` interface — expanded locals list plus raw opcode bytes
- `CustomSection` interface — named arbitrary byte payload
- `WasmModule` class — mutable container for all twelve section types,
  constructor initializes all fields to empty arrays and `start` to null
- 61 unit tests with 100% line, branch, function, and statement coverage
- Literate programming style: ASCII diagrams, binary encoding tables,
  worked examples, and truth tables inline with the code
