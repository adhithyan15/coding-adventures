## [0.1.0] - 2026-03-23

### Added

- `CodingAdventures::WasmLeb128::LEB128Error < StandardError` — raised on
  unterminated sequences or values exceeding the 32-bit range.
- `decode_unsigned(data, offset=0)` — decode a ULEB128-encoded unsigned integer
  from a binary String or Array of byte integers, returning
  `[value, bytes_consumed]`.
- `decode_signed(data, offset=0)` — decode a SLEB128-encoded signed integer
  using two's complement sign extension, returning `[value, bytes_consumed]`.
- `encode_unsigned(value)` — encode a non-negative integer as ULEB128,
  returning a binary String.
- `encode_signed(value)` — encode a signed integer as SLEB128,
  returning a binary String.
- Full test suite (32 tests, 103 assertions) covering all 11 specified cases:
  - Zero encoding/decoding
  - One-byte unsigned and signed values
  - One-byte signed negative (-2)
  - Multi-byte encoding (624485)
  - Max u32 (4294967295)
  - Max i32 (2147483647)
  - Min i32 (-2147483648)
  - Round-trip encode→decode for 14 unsigned values
  - Unterminated sequence error handling
  - Non-zero offset decoding
  - Round-trip for 15 signed values (including negatives)
  - Array and binary String input for decode functions
- Literate programming style: all source code annotated with diagrams,
  truth tables, worked examples, and step-by-step explanations.

### Notes

- Handles u32 / i32 range only (at most 5 LEB128 bytes). Ruby's arbitrary-
  precision integers require explicit 32-bit masking and sign-extension logic.
