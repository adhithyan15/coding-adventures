10
11 The renderer maps pixel-coordinate scenes to a fixed-width character grid.
12 Each cell in the grid is one character. The mapping uses a configurable
13 scale factor (default: 8px per char width, 16px per char height).
14
15 ```
16 Scene coordinates (pixels) Character grid
17 +---------------------+ +----------+
18 | rect at (0,0,200,32)| -> |##########|
19 | | |##########|
20 +---------------------+ +----------+
21 ```
22
23 ## Character Palette
24
25 Box-drawing characters create clean table grids:
26
27 ```
28 +------+-----+ Corners: top-left top-right bottom-left bottom-right
29 | Name | Age | Edges: horizontal vertical
30 +------+-----+ Tees: top-tee bottom-tee left-tee right-tee
31 | Alice| 30 | Cross: cross
32 +------+-----+ Fill: full-block
33 ```
34
35 ## Intersection Logic
36
37 When two drawing operations overlap at the same cell, the renderer
38 merges them into the correct junction character. A horizontal line
39 crossing a vertical line becomes a cross. A line meeting a box corner
40 becomes the appropriate tee.
41
42 This is tracked via a "tag" buffer parallel to the character buffer.
43 Each cell records which directions have lines passing through it
44 (up, down, left, right), and the tag is resolved to the correct
45 box-drawing character on each write.
46
47 ## Direction Bitmask
48
49 ```
50 UP (1)
51 |
52 LEFT(8)-+-RIGHT(2)
53 |
54 DOWN(4)
55 ```
56
57 The FILL flag (16) marks cells filled with block characters.
