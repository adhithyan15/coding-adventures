# Changelog

All notable changes to this package will be documented in this file.

## [0.2.0] - 2026-03-23

### Added

- Complete WASM 1.0 MVP opcode table: 172 instructions from 0x00 to 0xBF
- `@opcodes` module attribute — compile-time list of maps, zero runtime cost
- `@opcodes_by_byte` and `@opcodes_by_name` maps built from `@opcodes` at
  compile time via `Map.new/2`
- `get_opcode/1` — lookup by byte, returns `{:ok, map} | {:error, :unknown_opcode}`
- `get_opcode_by_name/1` — lookup by name, same return type
- `all_opcodes/0` — returns the full list of opcode maps
- 21 unit tests + 5 doctests covering: count, byte/name lookups, stack effects,
  immediates, uniqueness, category spot checks, required map keys, and store invariants
- Literate inline comments explaining: stack machine model, memarg format,
  structured control flow, signedness conventions, IEEE 754 notes, and
  type conversion semantics

### Notes

- The WASM 1.0 MVP has exactly 172 instructions; the "~183" figure sometimes
  cited includes post-MVP proposals that use a 0xFC prefix encoding
- All data is evaluated at compile time — `@opcodes` is a BEAM-level constant
