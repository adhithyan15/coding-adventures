# Changelog

All notable changes to this package will be documented in this file.

## [0.1.0] - 2026-03-23

### Added

- `LEB128Error` class extending `Error` — thrown on unterminated sequences or
  values exceeding the 32-bit range.
- `decodeUnsigned(data, offset?)` — decode a ULEB128-encoded unsigned integer
  from a `Uint8Array`, returning `[value, bytesConsumed]`.
- `decodeSigned(data, offset?)` — decode a SLEB128-encoded signed integer using
  two's complement sign extension, returning `[value, bytesConsumed]`.
- `encodeUnsigned(value)` — encode a non-negative integer as ULEB128,
  returning a `Uint8Array`.
- `encodeSigned(value)` — encode a signed integer as SLEB128,
  returning a `Uint8Array`.
- Full test suite (54 tests) covering all 11 specified test cases:
  - Zero encoding/decoding
  - One-byte unsigned and signed values
  - One-byte signed negative (-2)
  - Multi-byte encoding (624485)
  - Max u32 (4294967295)
  - Max i32 (2147483647)
  - Min i32 (-2147483648)
  - Round-trip encode→decode for 14 unsigned values
  - Unterminated sequence error handling
  - Non-zero offset decoding
  - Round-trip for 15 signed values (including negatives)
- 96% test coverage (lines), 95.83% branch coverage
- Literate programming style: all source code annotated with diagrams,
  truth tables, worked examples, and step-by-step explanations.

### Notes

- Handles u32 / i32 range only (up to 5 LEB128 bytes). i64/u64 requires
  BigInt and is outside the scope of WASM 1.0's primary use cases. Documented
  in source as a known limitation.
