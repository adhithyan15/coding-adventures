### Added

- Initial package scaffolding generated by scaffold-generator

## [0.1.1] - 2026-03-23

### Added

- Full implementation of the WASM 1.0 type system:
  - `ValueType` named byte type with constants: ValueTypeI32=0x7F, ValueTypeI64=0x7E, ValueTypeF32=0x7D, ValueTypeF64=0x7C
  - `ExternalKind` named byte type with constants: Function=0x00, Table=0x01, Memory=0x02, Global=0x03
  - `BlockType` named byte type with constant: BlockTypeEmpty=0x40
  - `ElementTypeFuncRef` byte constant (0x70 = funcref)
  - `FuncType` struct: Params and Results as ValueType slices
  - `Limits` struct: Min (required), Max (optional via HasMax bool)
  - `MemoryType` struct: wraps Limits
  - `TableType` struct: ElementType byte + Limits
  - `GlobalType` struct: ValueType + Mutable bool
  - `Import` struct: ModuleName, Name, Kind, TypeInfo (any for sum type)
  - `Export` struct: Name, Kind, Index
  - `Global` struct: GlobalType + InitExpr bytes
  - `Element` struct: TableIndex, OffsetExpr, FunctionIndices slice
  - `DataSegment` struct: MemoryIndex, OffsetExpr, Data bytes
  - `FunctionBody` struct: Locals ValueType slice, Code bytes
  - `CustomSection` struct: Name, Data bytes
  - `WasmModule` struct: all 12 section fields as slices + Start *uint32
- 45 unit tests covering all 21 spec test scenarios; passes go vet
- Literate programming style: ASCII diagrams, binary encoding tables, examples inline in godoc
