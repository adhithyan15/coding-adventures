4
5 ## What Is a Huffman Tree?
6
7 A Huffman tree is a full binary tree (every internal node has exactly two
8 children) built from a symbol alphabet so that each symbol gets a unique
9 variable-length bit code. Symbols that appear often get short codes; symbols
10 that appear rarely get long codes. The total bits needed to encode a message
11 is minimised — it is the theoretically optimal prefix-free code for a given
12 symbol frequency distribution.
13
14 Think of it like Morse code. In Morse, `E` is `.` (one dot) and `Z` is
15 `--..` (four symbols). The designers knew `E` is the most common letter in
16 English so they gave it the shortest code. Huffman's algorithm does this
17 automatically and optimally for any alphabet with any frequency distribution.
18
19 ## Algorithm: Greedy Construction via Min-Heap
20
21 1. Create one leaf node per distinct symbol, each with its frequency as its
22 weight. Push all leaves onto a min-heap keyed by weight.
23
24 2. While the heap has more than one node:
25 a. Pop the two nodes with the smallest weight.
26 b. Create a new internal node whose weight = sum of the two children.
27 c. Set left = the first popped node, right = the second popped node.
28 d. Push the new internal node back onto the heap.
29
30 3. The one remaining node is the root of the Huffman tree.
31
32 ## Tie-Breaking Rules (Deterministic Output)
33
34 Without tie-breaking, different implementations could build structurally
35 different trees from the same input — producing different (but equally valid)
36 code lengths. Deterministic tie-breaking ensures the canonical code table is
37 identical everywhere.
38
39 1. Lowest weight pops first.
40 2. Leaf nodes have higher priority than internal nodes at equal weight
41 ("leaf-before-internal" rule).
42 3. Among leaves of equal weight, lower symbol value wins.
43 4. Among internal nodes of equal weight, earlier-created node wins
44 (insertion-order FIFO).
45
46 ## Prefix-Free Property: Why It Works
47
48 In a Huffman tree:
49 - Symbols live ONLY at the leaves, never at internal nodes.
50 - The code for a symbol is the path from root to its leaf
51 (left edge = '0', right edge = '1').
52
53 Since one leaf is never an ancestor of another leaf, no code can be a prefix
54 of another code. This is the prefix-free property, and it means the bit
55 stream can be decoded unambiguously without separator characters: just walk
56 the tree bit by bit until you hit a leaf.
57
58 ## Canonical Codes (DEFLATE / zlib Style)
59
60 The standard tree-walk produces valid codes, but different tree shapes can
61 produce different codes for the same symbol lengths. Canonical codes
62 normalise this: given only the code *lengths*, you can reconstruct the exact
63 canonical code table without transmitting the tree structure.
64
65 Algorithm:
66 1. Collect (symbol, code_length) pairs from the tree.
67 2. Sort by (code_length, symbol_value).
68 3. Assign codes numerically:
69 code[0] = 0 (left-padded to length[0] bits)
70 code[i] = (code[i-1] + 1) << (length[i] - length[i-1])
71
72 This is exactly what DEFLATE uses: the compressed stream contains only the
73 length table, not the tree, saving space.
74
